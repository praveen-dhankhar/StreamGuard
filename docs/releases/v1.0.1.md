# StreamGuard v1.0.1

StreamGuard is a resilient Go reverse proxy for streaming LLM APIs. It protects client UX during mid-stream provider failure, exposes recovery behavior through explicit SSE gateway events, and keeps failed attempts out of billed usage when they are superseded.

## Highlights

- Deterministic failover across configured providers
- Explicit gateway SSE events for status, failover, regeneration, and truncation
- Per-key allowlists, admission-time rate limiting, and budget enforcement
- Usage ledger aggregation and reconciliation primitives
- Mock upstream harness for deterministic failure injection and local verification

## Demo

Run the local demo stack:

```sh
go run ./cmd/demo-upstreams
OPERATOR_TOKEN=dev-operator-token go run ./cmd/streamguard
go run ./client-ref --no-color --endpoint http://localhost:8080/v1/stream --api-key sg_live_demo "Explain failover behavior"
```

## Current Limitations

- Single-instance and in-memory only
- No persistent ledger or distributed breaker state
- No cross-provider model translation
- Calibration defaults are bootstrap values until replaced with measured production-like samples

## Scope Notes

This release is intentionally scoped as a single-instance, in-memory proxy focused on correctness of failover semantics, wire-protocol clarity, and usage accounting behavior.
